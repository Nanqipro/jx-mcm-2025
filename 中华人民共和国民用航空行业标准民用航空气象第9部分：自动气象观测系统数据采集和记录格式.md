4.1.1消息包含一个消息头域和数据部分，数据部分由1个或多个数据域构成。消息由ASCII字符串组成，用控制字符SOH、STX、ETX和EOT作为分隔符。

# 6数据部分  

# 6.1数据域  

6.1.1数据部分由一个或多个数据域构成，数据部分位于消息头域之后和消息结束分隔符（EOT）之前。数据部分内的每个数据域以STX开始，ETX结束，并且包含若干个子域。子域之间用分隔字符"|"(ASCII $0 \mathbf { x } 7 \mathbf { C } )$ 分隔。  
6.1.2表4定义了数据域的结构。  

表4数据域结构  


<html><body><table><tr><td>序号</td><td>子域和控制符</td><td>描述</td></tr><tr><td>1</td><td>STX</td><td>控制符，ASCII0x02，表示一个新数据域的开始。</td></tr><tr><td>2</td><td>Item name</td><td>数据项的名称，例如，“VERVIS”，“CH1INS”等。</td></tr><tr><td>3</td><td>Data type</td><td>数据类型：长整型（I），实型（R）或字符串型（S），见6.1.3。</td></tr><tr><td>4</td><td>Status</td><td>数据状态，有以下几种取值：N，M，C，O，一，I,U，见6.1.4。</td></tr><tr><td>5</td><td>Value</td><td>数据的值。</td></tr><tr><td>6</td><td>Unit</td><td>数据单位（例如“米”，“米每秒”等）。如果数据无单位，该项为空。所有单位均为国际制单位。</td></tr><tr><td>7</td><td>ETX</td><td>控制符，ASCII0x03，表示消息头域或数据域的结束。</td></tr></table></body></html>  

6.1.3数据的类型为以下几种：  

数据类型I（长整型）是一个可变宽度数字，为32位符号整型；  
数据类型R（实型）是一个可变宽度浮点数，在小数点左边应至少有一位，用“.”作为小数分隔符，小数点右边应至少有两位（例如1.23）；  
数据类型S（字符串）是一个可变宽度字符串。该字符串以分隔符“|”（ASCIIOx7C）结束，|”字符不应包含在字符串中。  

6.1.4数据的状态应使用一个状态编码来描述其有效性。数据的状态及其编码的定义见表5。  

表5数据状态编码

# 6.2数据更新时间  

6.2.1每条消息应包含表明数据对象被系统更新的时间，该时间为协调世界时，以Unixtime时间格式表示。更新时间作为一个数据域居于消息头域之后，其他数据域之前。  

示例：  

<SOH>1|00125|CL0UD|1|02L＜ETX><STX>TIME|I|N|1011098674| $<$ ETX><STx>otherfields.·<EOT>6.2.2时间数据域的“单位”子域为空。
